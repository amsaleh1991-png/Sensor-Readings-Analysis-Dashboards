# 📊 Sensor Readings Analysis Dashboard

## 📝 Project Overview
This project presents a comprehensive **Power BI Dashboard** designed to monitor and analyze industrial sensor data across multiple facilities, including **Plant A, Plant B, Plant C, and Warehouse 1**. The primary objective is to provide real-time visibility into environmental conditions and system health, enabling data-driven decisions for predictive maintenance and operational excellence. By tracking key performance indicators (KPIs) such as error rates and environmental fluctuations, this dashboard helps in identifying bottlenecks and preventing equipment downtime.

## 📈 Key Metrics & KPIs
The dashboard tracks several critical metrics to evaluate the performance and health of the sensor network. The following table summarizes the high-level KPIs extracted from the analysis:

| Metric | Value | Description |
| :--- | :--- | :--- |
| **Total Readings** | 3,000 | Total number of data points collected from all sensors. |
| **Error Rate** | 34.68% | Percentage of readings indicating a critical system error. |
| **Warning Rate** | 32.61% | Percentage of readings indicating a potential issue or warning. |
| **Avg. Temperature** | 18.84 °C | The mean temperature recorded across all monitored plants. |
| **Avg. Humidity** | 48.09% | The mean humidity level maintained within the facilities. |

## 🛠 Technical Stack
The development of this dashboard involved advanced data engineering and visualization techniques using the following tools:
*   **Power BI Desktop:** The primary tool for data visualization and dashboard construction.
*   **Power Query:** Utilized for data cleaning, transformation, and normalization of raw sensor logs.
*   **DAX (Data Analysis Expressions):** Employed to create complex measures for calculating dynamic error rates, warning percentages, and time-based averages.
*   **Data Modeling:** Established robust relationships between sensor metadata, plant locations, and temporal data to ensure accurate cross-filtering.

## 🔍 Data Insights
The analysis revealed several critical patterns that are essential for operational management:

| Category | Key Finding |
| :--- | :--- |
| **Plant Performance** | **Plant A** recorded the highest error count (256), indicating a need for immediate technical audit. |
| **Status Distribution** | The system health is split almost equally between **OK (32.72%)**, **ERROR (34.68%)**, and **WARN (32.61%)**. |
| **Seasonal Trends** | Temperature peaks in **June (21.1°C)**, while humidity reaches its lowest point in **October (45.0%)**. |
| **Critical Sensors** | Specific sensors like **SEN2181** and **SEN2407** consistently report multiple errors, marking them as high-risk components. |

## 💡 Recommendations
Based on the data insights, the following actions are recommended to improve system reliability:
1.  **Targeted Maintenance:** Prioritize technical inspections at **Plant A** and **Plant B** to address the high frequency of errors.
2.  **Sensor Replacement:** Conduct a deep dive into high-risk sensors (e.g., **SEN2181**) to determine if they require recalibration or full replacement.
3.  **Environmental Optimization:** Implement automated alerts for humidity drops in **October** to maintain optimal operating conditions.
4.  **Proactive Scheduling:** Use the identified temperature trends to schedule heavy-duty operations outside the peak heat month of **June**.
